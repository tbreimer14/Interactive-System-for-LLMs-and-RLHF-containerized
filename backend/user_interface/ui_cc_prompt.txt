Build me a **simple, modular user interface** for my local research project.

## Project context

This project is an interactive LLM system with these main parts:

1. **RAG system**
   - retrieves relevant chunks from a local document / subreddit-style knowledge base
   - passes retrieved context into the LLM prompt
   - generation and retrieval are already treated as separate modules

2. **Reward / feedback system**
   - the user can define their own reward traits
   - reward traits are **not hardcoded**
   - examples: clarity, empathy, humor, directness, caution, etc.
   - the user can score a response on multiple traits
   - these trait scores are later combined into a scalar reward like:
     `reward = w1*r1 + w2*r2 + w3*r3 ...`
   - weights are user-defined and should be editable in the UI

3. **PPO / RL fine-tuning later**
   - this does **not** need to be implemented now
   - but the UI must be designed so PPO can be connected later
   - the UI should support logging prompts, retrieved context, model outputs, trait scores, weights, and final scalar reward for future training

## Main goal

Create a **super simple frontend / local UI prototype** that lets a user:

- type a prompt
- send it to the backend
- see the generated response
- optionally inspect retrieved context from RAG
- define reward traits
- define weights for those traits
- score the model response on each trait
- see the computed scalar reward
- save the interaction to a local log for later RL / PPO use

This is for a **research prototype**, so prioritize:
- simplicity
- readability
- modularity
- easy future integration

Do **not** overengineer it.

## What I want built

Build a minimal UI with these sections:

### 1. Prompt input area
A simple text box where the user can enter a prompt, like a subreddit-style post or question.

Buttons:
- **Generate Response**
- **Clear**

### 2. Response display area
Show:
- the model response
- optionally the final prompt sent to the LLM
- a loading state while generation is running

### 3. Retrieved context panel
A collapsible section that shows:
- top-k retrieved chunks
- chunk source / metadata if available
- keep this read-only

This is just for transparency / debugging.

### 4. Reward traits editor
A simple section where the user can:
- add a trait name
- remove a trait
- assign a weight to each trait

Example:
- clarity: weight 1.0
- empathy: weight 0.7
- humor: weight 0.2

Traits must be dynamic, not hardcoded.

### 5. Response scoring panel
After a response is generated, the user should be able to score that response for each current trait.

For each trait:
- numeric score input, e.g. slider or small number box
- simple range like 0 to 5 or -5 to 5

Then compute:

`scalar_reward = sum(weight_i * score_i)`

Display:
- each trait score
- each weight
- weighted contribution
- final scalar reward

### 6. Save interaction button
A button to save the full interaction locally.

Each saved interaction should include:
- user prompt
- retrieved chunks
- model response
- trait names
- trait weights
- trait scores
- scalar reward
- timestamp

Store this in a simple local format like JSONL.

### 7. History panel
A simple side panel or bottom section showing past saved interactions.

For each saved item show:
- timestamp
- shortened prompt
- shortened response
- scalar reward

Optional:
- click one to reopen details
