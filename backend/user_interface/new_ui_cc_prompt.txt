Build me a **simple, modular user interface** for my local LLM research project.

==================================================
PROJECT CONTEXT
==================================================

This system has 3 parts:

1. RAG system
- retrieves relevant chunks from local data
- builds prompt using retrieved context
- generates response
- retrieval and generation are separate modules

2. Reward / feedback system
- user defines reward traits (NOT hardcoded)
- examples: clarity, empathy, humor, directness
- each trait has:
    - weight
    - score (given after seeing response)
- scalar reward:
    reward = sum(weight_i * score_i)

3. PPO later (NOT NOW)
- do NOT implement PPO
- UI must support logging for future RL training

==================================================
MAIN GOAL
==================================================

Create a **super simple UI prototype** that allows a user to:

- enter a prompt
- generate a response
- view retrieved context (RAG)
- define reward traits
- assign weights to traits
- score the response
- compute scalar reward
- save interaction
- view history

Keep it:
- simple
- clean
- modular
- easy to expand

DO NOT overengineer.

UI SECTIONS
==================================================

1. PROMPT INPUT

    - Clear

--------------------------------------------------

2. RESPONSE DISPLAY
Show:
- model response
- optional final prompt
- loading indicator during generation

--------------------------------------------------

3. RETRIEVED CONTEXT PANEL
- collapsible section
- shows top-k retrieved chunks
- show:
    - chunk text
    - optional source

Read-only.

--------------------------------------------------

4. REWARD TRAITS EDITOR

User can:
- add trait
- remove trait
- edit weight

Example:
- clarity: 1.0
- empathy: 0.7
- humor: 0.2

Traits must be dynamic (NOT hardcoded).

--------------------------------------------------

5. RESPONSE SCORING PANEL

After generation:
- for each trait:
    - input score (slider or number)
    - range: 0–5 or -5–5

Compute:
- contribution = weight * score
- scalar reward = sum(contributions)

Display:
- trait scores
- weights
- contributions
- final scalar reward

--------------------------------------------------

6. SAVE INTERACTION

Button:
- Save Interaction

Save:
- prompt
- retrieved chunks
- response
- traits (name, weight, score, contribution)
- scalar reward
- timestamp

Store locally as JSONL.

--------------------------------------------------

7. HISTORY PANEL

Show past interactions:
- timestamp
- short prompt
- short response
- scalar reward

Optional:
- click to expand details
