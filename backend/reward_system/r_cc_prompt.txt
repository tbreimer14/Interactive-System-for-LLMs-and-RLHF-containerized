Build me a **super simple, modular local reward system prototype** in Python for a research project.

## Goal

Create the **simplest implementation** of a reward system where the **user defines their own reward traits**, so that later it can be connected to:

- a frontend / UI
- a text generation module
- a logging / analytics dashboard
- PPO or another RL fine-tuning loop

Do **not** overengineer it.
Do **not** build PPO yet.
Do **not** build the frontend yet.
I only want the reward system as a separate module that can be joined later.

---

## Core idea

The system should work like this:

1. Load a user-defined trait configuration from JSON
2. Accept:
   - a user prompt
   - a generated model response
3. For each trait, score the response based on the trait's description
4. Combine the trait scores using user-defined weights
5. Return:
   - each trait score
   - the final scalar reward

The final reward should be a weighted sum:

reward = sum(weight_i * score_i)

---

## Very important design constraints

- The traits must **not be hardcoded**
- The user should be able to define traits like:
  - interactivity
  - warmth
  - directness
  - humor
  - formality
  - empathy
  - etc.
- The system must be modular and easy to swap into a larger project later
- Keep files small and readable
- Use clear function boundaries
- Add comments so I can understand the code

---

## Simplest implementation choice

For now, do **not** use a real LLM judge API.
Instead, make a **mock / placeholder trait scorer** so the system runs locally and clearly.

That means:

- create a trait scoring function that returns simple dummy scores for now
- structure the code so I can later replace that scorer with:
  - an LLM judge
  - a classifier
  - a human feedback scorer

The point is to build the architecture now, not the final scoring intelligence.

## File responsibilities

### traits.json
Store user-defined traits in this format:

```json
{
  "traits": [
    {
      "name": "interactivity",
      "description": "Encourages continued conversation and asks useful follow-up questions",
      "weight": 0.5
    },
    {
      "name": "warmth",
      "description": "Sounds kind, supportive, and human",
      "weight": 0.3
    },
    {
      "name": "directness",
      "description": "Gets to the point without being vague",
      "weight": 0.2
    }
  ]
}
