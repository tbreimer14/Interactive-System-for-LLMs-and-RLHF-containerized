Build me a **super simple, modular local prototype** for my research project in Python.

## Project goal

I am building a system where:

1. a language model generates responses
2. the **user defines their own reward traits**
3. the **user scores model responses on those traits**
4. the system converts those scores into **one scalar reward**
5. that scalar reward is used later for **PPO / RL fine-tuning**

The two most important parts of the project are:

- **human feedback / user scoring**
- **PPO later**

Do **not** overengineer this.
Do **not** build PPO yet.
Do **not** build a polished frontend yet.
I want the **simplest clean architecture** that can be joined together later.

---

## Very important project direction

This project should **NOT** hardcode reward traits like:
- toxicity
- helpfulness
- warmth
- empathy

Instead, the **user must define their own traits**.

Example:
- interactivity
- directness
- humor
- formality
- kindness
- creativity
- or anything else the user wants

So the reward system must be **trait-agnostic** and **user-configurable**.

Build the **smallest clean local implementation** of these pieces:

---

### 2. Response evaluation / scoring pipeline
The system should accept:
- a prompt
- a generated response
- user-defined traits

Then it should allow **user-provided trait scores** for that response.

Example user feedback:

```json
{
  "interactivity": 5,
  "warmth": 3,
  "directness": 4
}
```

Then compute:

```python
reward = sum(weight_i * user_score_i)
```

This final scalar reward is what PPO will use later.

## Important design rules

### Rule 1: No hardcoded reward traits
The code must not assume fixed traits like helpfulness/toxicity/etc.

### Rule 2: Keep trait scoring external
The reward function should not “decide” the scores itself.
For now, the **user provides the scores**.

That means the reward system is basically:

```text
prompt + response + traits + user_scores -> final scalar reward
```

### Rule 3: PPO compatibility
Design the system so PPO can later consume:
- prompt
- response
- scalar reward

### Rule 4: Modular integration later
This reward system must be easy to connect later to:
- a frontend
- a generation system
- a PPO trainer
- a visual analytics dashboard
